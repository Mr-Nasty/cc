STEP 1: Create a VPC

Login to AWS Console.

Go to VPC Dashboard.

Click "Create VPC".

Select "VPC only".

Enter the following details:
Name: MyVPC
IPv4 CIDR block: 10.0.0.0/16

Click Create VPC.

STEP 2: Create Public Subnet

Go to Subnets.

Click Create Subnet.

Select the VPC you created.

Enter:
Subnet Name: PublicSubnet
CIDR Block: 10.0.1.0/24

Click Create Subnet.

STEP 3: Create Private Subnet

Click Create Subnet again.

Select the same VPC.

Enter:
Subnet Name: PrivateSubnet
CIDR Block: 10.0.2.0/24

Click Create Subnet.

STEP 4: Create Route Tables

Public Route Table

Go to Route Tables.

Click Create Route Table.

Name: PublicRT

Select your VPC.

Click Create.

Private Route Table

Create another Route Table.

Name: PrivateRT

Select the same VPC.

Click Create.

STEP 5: Associate Subnets with Route Tables

Public Route Table

Select PublicRT.

Go to Subnet Associations.

Click Edit Associations.

Select PublicSubnet.

Private Route Table

Select PrivateRT.

Edit Subnet Associations.

Select PrivateSubnet.

STEP 6: Create Internet Gateway

Go to Internet Gateways.

Click Create Internet Gateway.

Name: MyIGW.

Click Create.

Attach Internet Gateway to VPC

Select the Internet Gateway.

Click Attach to VPC.

Choose your VPC.

STEP 7: Create NAT Gateway

Go to NAT Gateways.

Click Create NAT Gateway.

Select PublicSubnet.

Allocate Elastic IP.

Click Create NAT Gateway.

STEP 8: Update Public Route Table

Open PublicRT.

Go to Routes.

Click Edit Routes.

Add Route:

Destination: 0.0.0.0/0
Target: Internet Gateway

Save changes.

STEP 9: Update Private Route Table

Open PrivateRT.

Go to Routes.

Click Edit Routes.

Add Route:

Destination: 0.0.0.0/0
Target: NAT Gateway

Save changes.

STEP 10: Create Security Groups

Public Security Group

Go to Security Groups.

Click Create Security Group.

Name: PublicSG.

Add Inbound Rule:

Type: SSH
Source: My IP

Save.

Private Security Group

Create another Security Group.

Name: PrivateSG.

Add Inbound Rule:

Type: SSH
Source: Public Instance

STEP 11: Create Key Pair

Go to EC2 Dashboard.

Click Key Pairs.

Click Create Key Pair.

Name: vpc-kp.

Download the .pem file.

STEP 12: Create IAM Role

Go to IAM Dashboard.

Click Roles.

Click Create Role.

Select EC2 as trusted entity.

Attach required permissions.

Create the role.

STEP 13: Launch Public EC2 Instance

Go to EC2 → Launch Instance.

Name: Public-Instance.

Choose Ubuntu AMI.

Instance Type: t2.micro.

Select Key Pair: vpc-kp.

Network Settings:
VPC: MyVPC
Subnet: PublicSubnet

Enable Auto Assign Public IP.

Select Security Group: PublicSG.

Launch Instance.

STEP 14: Launch Private EC2 Instance

Launch another EC2 instance.

Name: Private-Instance.

VPC: MyVPC.

Subnet: PrivateSubnet.

Disable Public IP.

Select Security Group: PrivateSG.

Launch Instance.

STEP 15: Connect to Public EC2 Instance (SSH)

Open Command Prompt and run:

icacls vpc-kp.pem /inheritance:r

icacls vpc-kp.pem /grant user-20:r

Then connect using SSH:

ssh -i "vpc-kp.pem" ubuntu@PUBLIC-IP

Example:

ssh -i "vpc-kp.pem" ubuntu@13.126.101.78

STEP 16: Copy File Using SCP

To transfer file from local system to EC2 instance:

scp -i "vpc-kp.pem" vpc-kp.pem ubuntu@PUBLIC-IP:/home/ubuntu

chmod 400 vpc-kp.pem

ssh -i vpc-kp.pem ubuntu@10.0.0.110 (Private IP)
